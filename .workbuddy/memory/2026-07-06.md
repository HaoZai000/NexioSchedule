# 2026-07-06 工作日志

## 搭配页面课程卡片模糊 bug 修复
- **问题1**：在搭配页面调整一个搭配的课程卡片模糊后，切换到其他搭配时模糊值跟随变动
- **根因**：`CustomizeScheduleScreen.kt` 第 171 行 `effectValue` 使用无 key 的 `remember`，切换搭配时不重置，导致滑块保留上一个搭配的模糊值，交互后错误应用到新搭配
- **修复**：`remember` 改为 `remember(currentCombinationIndex)`，切换搭配时自动用 `initialCardBlurRadius` 重新初始化

- **问题2**：调整模糊后卡片预览快照不更新，切搭配后其他卡片也不反映各自的模糊值
- **根因**：`onEffectValueChange` 只更新 `courseCardBlur` 和 `cardBlurRadius`，未重新捕获 `customizeSnapshot` 和 `combinations[idx].snapshot`
- **修复**：在 `onEffectValueChange` 中加入 150ms 防抖的快照重新捕获逻辑，同时更新 `customizeSnapshot`（当前卡片实时预览）和 `combinations[idx].snapshot`（切搭配后该卡片预览）；切换搭配时取消防抖任务避免竞态

- **问题3**：进入搭配页左滑，下一个搭配卡片先显示当前搭配模糊程度的快照，再更新为正确的
- **根因**：进入搭配页时预捕获相邻搭配快照（line 1299-1311），只切换了 wallpaper bitmap/offset/scale，**漏切 `courseCardBlur`**，导致快照用错误模糊值捕获
- **修复**：预捕获前加 `courseCardBlur = nextComb.cardBlurRadius`，捕获后恢复 `savedBlur`

- **问题4**：模糊设为 0 时课程卡片背景颜色遮罩消失
- **根因**：`CourseCard.kt` 中 `cardColor`（半透明颜色遮罩）仅通过 blur 的 `blendColors` 应用；blur=0 时 `hasBlur=false`，跳过 `textureBlur`，Card 颜色为 `Color.Transparent`，遮罩丢失
- **修复**：Card 颜色改为 `if (hasBlur) Color.Transparent else cardColor`，无 blur 时回退到纯色遮罩

- **问题5**：点击返回时主界面模糊不立刻还原，要等动画结束
- **根因**：`dismissCustomize` 只还原了壁纸，`courseCardBlur` 在动画结束后才还原
- **修复**：`dismissCustomize` 中和壁纸一起立刻 `courseCardBlur = originalCourseCardBlur`，并取消防抖任务

- **问题6**：在开洞编辑模式调了模糊后点取消，模糊没有还原（只有壁纸还原了）
- **根因**：`onRevertWallpaper` 只还原壁纸 bitmap/offset/scale，没有还原 `courseCardBlur`；缺少 `savedCourseCardBlur` 变量
- **修复**：新增 `savedCourseCardBlur` 变量，在所有设置 `savedWallpaperBitmap` 的位置同步设置；`onRevertWallpaper` 中增加 `courseCardBlur = savedCourseCardBlur` 并取消防抖任务

- **问题7**：第一次进入搭配页，自定义当前搭配时效果滑块显示的是下一个搭配的模糊值
- **根因**：进入搭配页时预捕获下一个搭配快照，临时将 `courseCardBlur` 改为下一个搭配的模糊值；`CustomizeScheduleScreen` 的 `effectValue` 通过 `initialCardBlurRadius = courseCardBlur` 初始化，若组合发生在预捕获改值之后，`effectValue` 就被初始化为错误值
- **修复**：`initialCardBlurRadius` 改为 `combinations.getOrNull(currentCombinationIndex)?.cardBlurRadius ?: 0f`，直接读组合数据而非可能被预捕获临时修改的 `courseCardBlur`
